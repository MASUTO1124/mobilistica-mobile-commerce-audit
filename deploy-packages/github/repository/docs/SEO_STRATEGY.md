## 1. 基本方針

記事量産ではなく、**ツール（診断エンジン）主導のブランド獲得・被リンク獲得**を軸とする。診断ツール自体が「試して価値がわかる」資産であるため、記事はツールへの導線・補完説明という位置づけとし、SEO Growth Engine配下の他サイトのような高頻度記事量産運用は本プロジェクトには適用しない。

- **ピラー（軸ページ）**: ランディングページ `/tools/mobile-commerce-audit/`（landing-page/仕様。web-wp-spec.md準拠）
- **事例（社会的証明）**: post-635「ECサイトのモバイルPageSpeedスコアを28点から76点に改善した実測記録」（2026-07-14公開・現時点でMobilistica唯一の公開記事）
  - **注意**: 28→76点の実測値は別施策（psi-optimize.php等の改善実施）の実績であり、本診断ツールの効果として記述しない（PROJECT_BRIEF.md 安全ルール）。事例参照・信頼性訴求としてのみリンクする

## 3. 執筆前の必須プロセス

**すべてのSEOクラスター記事は、執筆前に以下2点の確認を必須とする**（docs-suite-spec.md明記）。

1. `cannibal_check --pre` の実行（既存記事とのキーワード・意図の重複確認）
2. SEO Growth Engineバックログとの照合（`~/.claude/skills/seo-growth-engine/` 配下の既存Opportunity・記事クラスタ優先度との重複・競合チェック）

この2点を経ずに記事を執筆・公開しない。Mobilisticaサイトは公開記事が現時点で1本のみ（post-635）であり、カニバリゼーションのリスクは現状低いが、記事数が増えるにつれてリスクが高まるため、プロセスとして固定する。

## 4. 記事とツールの関係

- 記事はLP（ピラー）およびツール本体（Web診断UI）への導線を必ず含める
- 記事単体でのキーワード上位表示を最終目標とせず、「診断ツールを使ってもらう」ことを一次KPIとする（詳細な計測方法はMEASUREMENT_PLAN.md参照）
- P1クラスターは特に「診断結果の読み方」など、ツール利用後の理解を助ける補完コンテンツとしての役割を重視する
